subject Constant folder removes public methods from objects
code 
object M {
  def f = 3;
}
what happened 
I got an empty object "M", as the "f" method has been constant-folded.
what expected

I expected an object "M" accessible from Java, with a method called "f".

It is my opinion that changing the public interface of an object is a bad idea, because it changes the way this object is viewed from Java. But since we haven't documented (I believe) how our classes look from Java, that might be no big deal. I'll let you decide.
