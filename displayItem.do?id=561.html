		what happened
		
The package of ScalaGui version 0.1.1 has a src directory tree with messed-up filenames.  There are literal \ ch\
aracters in the zip file, and sbaz obligingly creates filenames with \'s in them instead of creating subdirector\
ies.


	

	

		what expected
		Maybe the command-line tool should replace \'s by /'s automatically when it unpacks an sbp.  After all, it is supposed to run on Windows, where literal \ characters are impossible anyway.  Probably also, though, the sbaz ant task should not insert such characters to begin with.
