subject type checker complains about "viewable" when it shouldn't
code 
/* The following code should provide a view from List[A] to List[B], provided A <% B. */

object Main with Application {

  def view[B, A <% B](l: List[A]): List[B] = l match {
    case Nil => Nil;
    case x :: xs => (x: B) :: Main.this.view(xs);
  }
}
what happened 
v.scala:16: type instantiation with [B,A] failed since A is not viewable as B
    case x :: xs => (x: B) :: Main.this.view(xs);
                                       ^
what expected No error. A is viewable as B.
